个人需求

考虑到平常会切换工作平台 时而Windows 时而Mac, 时而Cubase 时而Studio one, 所以我急需一个能跨平台的音源 能让我的工程在任意平台无缝切换

Kontakt入库相关问题

  1. 加载乐器时报错提示:This instrument belongs to a library that is currently not installed.

    这主要是由于将标准音色库当成非标准音色库入库导致的, 如果画蛇添足给标准音色库添加了nicnt文件,那么就会报错

  2. 为什么有些标准音色库中包含nicnt文件, 也就是说按照非标准入库后却不报错

    那这有可能是nicnt文件中的SNPID值并非十六进制形式,使得入库工具未能识别,依然将其当做标准音色库来入库, 如下图:

    假如我们将该值修改成入库工具能识别的十六进制形式,比如E332, 那么标准音色库被当成非标准音色库入库,则报错This instrument belongs to a library that is currently not installed.

  3. 标准音色库使用工具入库后,kontakt中不显示

    这应该是kontakt版本过低导致的,无法识别二代音色库, 建议升级Kontakt

编曲中和弦的编配

发表于 2022-08-22

前言

前面给大家介绍过如何给旋律编配和弦, 详情参见《给旋律配和弦的几种方式》

也就是先有旋律 后有和弦

那如果没有旋律呢 比如我们需要写一段配乐 烘托某个场景的氛围 这种情况 我们通常会采用和弦先行的做法, 也就是先编配一串符合情绪的和弦

我们知道 不同的和弦连接 营造的情绪氛围是不一样的, 前面的文章也有提到过这一块 详见《和弦走向影响歌曲的感情色彩 节奏决定了歌曲的律动》

我们在编曲的时候 假如我们第一件事要做的是和弦连接的选择, 那就需要我们用耳朵去听 用心去感受 体会其中蕴含的情绪

那么具体该如何操作呢, 在这里 给大家介绍下我自己的工作流程, 仅供参考:

低音试探

我首先会先用低音旋律来试探情绪走向, 从大体听感上来讲, 和弦进行其实相当于是一段旋律 那么 我们可以将低音旋律理解为简化版的和弦进行

借此 我们可以大体勾勒出和弦的走向 这部分线条我很有可能直接就给贝斯演奏了

构建和弦

有了低音后 我们以此为根音 构建和弦

和弦大体分为六种, 大小增减属中, 具体参见《和弦的解析》

这一步需要我们进行不同种类的尝试 我们只需要使其听感还原或者接近低音旋律的效果即可

总结

以上这种方法算不上有多效率, 只能算是众多方法中的笨方法, 此法可以巩固我们对于和弦的理解和掌握, 另外还可以摆脱常用和弦套路的影响, 让声音回归到耳朵上, 这也算是它的优点了

我平常闲来无事时 就会用这种方法来写写自己的当下的情绪 然后 觉得还不错~

关于和弦编配 实际上自由度很大, 你想让歌曲按照怎样的情绪走 那就怎样编配 规矩是一方面 突破规矩也许能迎来新的世界

思考

我们在写歌的时候 可能会借助乐器 比如吉他或者钢琴 一边弹和弦一边哼唱

在这种情况下 和弦的挑选就显得尤为关键了 哼唱的旋律 很大程度是取决于和弦的

和弦在前 旋律在后 也就是和弦声音先出来 然后哼唱紧随

这一特性 我们可以运用到编曲中, 比如和弦演奏加花 使得接下来出现的旋律更加自然和顺畅连贯

  • 鼓组贝斯: 常规编排 小间奏在重拍演奏后立马停止 使情绪冷却下来

发表于 2022-07-02

前言

《FLY》一首非常不错的藏语流行歌 地铁上耳机回听, 一时兴起, 对该歌的编曲进行简单分析, 供日后编曲参考借鉴

歌曲试听

乐器架构

该歌曲用到的乐器主要有以下几类, 仅仅是类别, 并不指代具体某个乐器, 比如铺底乐器你可以使用电钢或者合成音色, 取决于自己

以下只介绍编排手法:

  • 鼓组贝斯: 先是每拍底鼓 然后追加踩镲 紧接着二四拍换成军鼓 两个军鼓前后采用不同音色处理
  • 铺底乐器: 柱式和弦每小节弹一次 贯穿全曲
  • 律动乐器: 按照一定的节奏对和弦进行有序分解 每小节变换一次和弦
  • 氛围乐器: 放置于左右两边扩大声场 副歌开始出现 营造氛围
  • 间奏乐器: 间奏部分充当主奏乐器演奏旋律, 间奏过后声场往后移变成持续律动乐器, 演奏的依然是间奏旋律
